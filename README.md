## Acerca del proyecto

Proyecto para un CRUD (Create, Read, Update, Delete) de pedidos y cuentas. Utilizo el patrón de diseño Repository que se utiliza para separar la lógica del modelo de la capa de persistencia de datos, lo que permite una mayor flexibilidad y escalabilidad.
